# 研报精读工作流

## PDF 精读要求

- PDF 精读模式由 `.env` 的 `FXBAOGAO_USE_PDF_READING` 控制，默认关闭。
- 未开启时，可以基于 detail 接口摘要、目录和正文摘录完成精读底稿、事实卡片和后续分析。
- 可以用 detail 接口信息筛选精读报告、登记元数据、定位目录或页码线索。
- 开启时，必须先打开 `intermediate/downloads/` 下的 PDF 原文，再填写 `REPORT_READER_精读分析.md`。
- 开启时，detail 接口返回的摘要、AI 总结、目录和正文摘录不得直接作为核心判断、关键数据或事实卡片来源。
- 开启时，每个关键事实必须记录 PDF 页码或图表编号；无法定位页码的内容只能作为待核对项。

## 单篇报告

按三层阅读法处理：

### 第一层：核心逻辑

找出：

- 报告标题
- 报告类型
- 核心观点
- 投资建议

回答一个问题：这篇报告到底在押什么判断。

### 第二层：假设与证据

重点抽：

- 关键假设
- 核心数据
- 主要证据
- 风险提示

如果风险提示明显弱于正文叙事，要单独指出。

### 第三层：深度判断

评估：

- 有没有一手数据
- 有没有明确口径
- 财务或行业假设是否激进
- 哪些内容只是叙事，不是证据

## 多篇报告

如果用户给出多篇报告，至少做三件事：

1. 共识提炼
2. 分歧定位
3. 推荐最值得精读的 1 到 2 篇

## 推荐输出骨架

### 报告摘要

- 类型
- 机构与日期
- 核心观点
- 关键假设
- 风险提示
- 一句话判断

### 跨报告对比

- 一致性信号
- 矛盾点
- 预期差

### 后续研究建议

- 还需要补什么外部资料
- 先验证哪些数字和假设
