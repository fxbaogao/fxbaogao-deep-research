# fxbaogao-deep-research

一个 Codex/Claude Skill，通过“先想后搜”的系统化方法，把行业、公司或主题研究转化为可追溯、可复核、可交付的研报深度研究工作区。

本项目专门对接发现报告（fxbaogao）接口，支持研报搜索、段落拉取、PDF 下载地址获取、核心材料筛选、事实卡片沉淀和最终调研报告生成。

## 痛点

在用研报做行业或公司研究时，你经常面临：

- **信息过载**：候选研报数量多，标题相近，难以快速筛出核心材料
- **检索粗放**：只搜一个宽泛关键词，容易遗漏产业链、竞争格局、政策、需求等关键维度
- **来源混乱**：结论来自哪篇报告、哪个报告 ID、哪个机构不清楚
- **事实不可追溯**：关键数据和判断没有事实卡片承接，后续难以复核
- **PDF 与摘要混用**：paragraph 摘要、正文摘录和 PDF 原文边界不清，容易把摘要当作精读
- **交付成本高**：从搜索结果到可读报告，中间缺少稳定工作流和模板

## 解决方案

fxbaogao-deep-research 提供一套面向发现报告的研报研究流程：

- **先想后搜**：先定义研究边界，再用 MECE 拆解 3-7 个关键子问题
- **按子问题定向检索**：每个子问题都有独立关键词，避免只依赖宽泛主题词
- **候选报告手工归并**：按子问题记录搜索结果，合并重复报告
- **核心报告筛选**：按相关性、时效性、深度和机构权威性筛选
- **PDF 原文核对**：正式研究或引用原文时，下载 PDF 并基于原文复核
- **事实卡片机制**：每条关键事实标注来源、报告 ID 和可信度
- **显式推导链**：通过“事实卡片 -> 对比框架 -> 推导过程”形成结论
- **可交付输出**：使用 Markdown 模板沉淀中间产物和 `FINAL_调研报告.md`

## 工作流程（9 步法）

```text
Step 1: 定义研究主题和边界
Step 2: 按 MECE 原则拆解 3-7 个子问题
Step 3: 按子问题搜索候选研报
Step 4: 筛选核心报告，区分核心必读、补充参考、背景材料
Step 5: 拉取核心报告段落，准备研究工作区
Step 6: 精读报告，填写 REPORT_READER_精读分析.md
Step 7: 沉淀事实卡片，标注来源、可信度、适用范围
Step 8: 建立对比框架与推导链
Step 9: 汇总输出 FINAL_调研报告.md
```

硬性规则：调用搜索、段落或下载接口前，必须先产出可审阅的研究边界和 MECE 子问题搜索矩阵。

## 工作区结构

主题研究会在 `workspace/` 下生成工作区：

```text
workspace/topic-<主题>-<日期>/
├── REPORT_READER_精读分析.md
├── FINAL_调研报告.md
└── intermediate/
    ├── 00_问题拆解.md
    ├── 01_资料来源.md
    ├── 02_事实卡片.md
    ├── 03_对比框架.md
    └── 04_推导过程.md
```

模板位于 `assets/workspace_templates/`。

其中：

- `00_问题拆解.md`：研究边界、核心问题、MECE 子问题和搜索关键词矩阵
- `01_资料来源.md`：候选研报、核心报告筛选记录和阅读链接
- `02_事实卡片.md`：事实、来源、可信度沉淀
- `03_对比框架.md`：不同报告对同一子问题的框架、假设、结论对比
- `04_推导过程.md`：从事实到结论的显式推理链

## 核心特性

| **定向检索** | 每个子问题绑定关键词，保留搜索维度映射 |

| **事实卡片** | 每条事实都标注来源、报告 ID 和可信度 |
| **显式推导** | 通过对比框架和推导过程形成可复核结论 |
| **Markdown 工作区** | 使用模板沉淀中间产物和最终报告 |

## PDF 精读

默认情况下，轻量研究可以使用 paragraph 接口返回的摘要和正文摘录。正式研究、需要引用原文或需要更高可复核性时，应通过下载接口获取 PDF 地址，并基于 PDF 原文核对。

不要把 paragraph 摘要或正文摘录伪装成 PDF 原文精读。

## 常见问题

**Q: 这和直接搜索发现报告有什么区别？**

A: 本项目不是只返回搜索结果，而是强制先拆解研究问题，再按子问题检索、筛选核心报告、沉淀事实卡片，并输出可追溯的研究过程。

**Q: 没有 API key 能用吗？**

A: 不能使用。搜索、段落和下载接口都需要有效的 `FXBAOGAO_API_KEY`。

**Q: paragraph 模式和 PDF 精读怎么选？**

A: 快速初筛或轻量研究可以用 paragraph 接口；正式研究、需要引用原文或需要更高可复核性时，用 PDF 原文。

**Q: 可以只研究一篇报告吗？**

A: 可以。直接调用段落接口，按 `REPORT_READER_精读分析.md` 模板精读即可。
