# 发现报告深度研究 Skill

## 何时使用

- 用户要研究行业、公司或主题，并需要系统收集研报与结构化分析。
- 用户希望从发现报告中筛选核心观点、关键事实、数据口径和分歧。
- 用户明确要求基于发现报告、fxbaogao、研报、报告 ID 或 PDF 做研究。

## 先读什么

- 调用任何发现报告接口前，先读 `references/api-usage.md`。
- 做多篇报告的完整深度研究时，读 `references/research-playbook.md`。
- 做单篇报告精读时，读 `references/report-reader-playbook.md`。
- 需要落盘时，复用 `assets/workspace_templates/` 下的模板结构。

## 路由规则

### 单篇报告精读

当用户给出明确报告 ID、单篇 PDF、单篇报告标题，或明确要求“精读某一篇报告”时，直接进入单篇精读流程。

- 不默认创建完整多篇研究工作区。
- 不要求先做 MECE 子问题搜索矩阵。
- 不额外搜索候选研报，除非用户要求补充验证。
- 输出围绕该报告：基础信息、核心逻辑、关键证据、假设、风险、事实卡片、待外部验证点、精读判断。

### 多篇深度研究

当用户提出行业、公司或主题研究时，使用完整研究流程：

1. 定义研究主题和边界。
2. 拆解 3-7 个 MECE 子问题，并为每个子问题给出搜索关键词。
3. 按子问题搜索候选研报。
4. 筛选核心必读、补充参考和背景材料。
5. 拉取核心报告段落；需要引用原文或精读时下载 PDF 核对。
6. 精读核心报告，沉淀事实卡片。
7. 建立对比框架和推导过程。
8. 输出最终调研报告。

硬性门槛：调用搜索接口前，必须先产出可审阅的研究边界和 MECE 子问题搜索矩阵。如果用户没有给出边界，先做保守默认并写明；只有边界会显著改变结论时才追问。

## 工作区

完整研究需要落盘时，创建：

```text
workspace/topic-<主题>-<日期>/
├── REPORT_READER_精读分析.md
├── FINAL_调研报告.md
└── intermediate/
    ├── 00_问题拆解.md
    ├── 01_资料来源.md
    ├── 02_事实卡片.md
    ├── 03_对比框架.md
    └── 04_推导过程.md
```

模板位于 `assets/workspace_templates/`。文件内容规范和研究细节见 `references/research-playbook.md`。

## 接口与证据约束

- 搜索、段落、下载接口以 `references/api-usage.md` 为准。
- 发现报告资料必须可追溯到报告 ID 和官网阅读链接：`https://www.fxbaogao.com/view?id=<reportId>`。
- paragraph 回包只适合快速初筛和轻量研究。深度研究、原文引用、关键数据核验必须下载 PDF 并基于 PDF 原文核对。
- 不伪造作者、机构、日期、摘要、页码或数据口径。
- 段落查询回答优先提供总结与 key takeaways，不整段复制正文。
- 网络错误时提示用户需允许访问 `api.fxbaogao.com`；下载失败时检查 `https://dr.fxbaogao.com/` 拼接后的 PDF 地址。

## 回复约定

- 多篇深度研究的第一条实质回复必须包含：研究边界、核心问题、3-7 个 MECE 子问题、搜索关键词矩阵。
- 搜索结果按与子问题的相关性分层呈现，不逐篇倾倒。
- 最终报告必须包含：研究边界、子问题回顾、核心发现、证据链、关键风险、不确定性、下一步方向和参考资料。
- 参考资料使用 Markdown 链接：`[报告ID - 报告标题](https://www.fxbaogao.com/view?id=报告ID)`。
